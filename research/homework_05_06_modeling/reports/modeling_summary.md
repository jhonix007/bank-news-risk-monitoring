# Краткий отчёт по ML-моделированию

## 1. Постановка задачи

Основная задача текущей clean-версии ДЗ №5-6 — binary classification по target `alert_flag`.

```text
title + text_fragment + entity_norm -> alert_flag
```

Модель должна определить, является ли новостной фрагмент риск-сигналом для банка. Положительный класс `alert_flag = 1` соответствует фрагментам, которые потенциально требуют внимания аналитика.

Поле `risk_type_4cls` остаётся дополнительной разметкой, описывающей тип риска. В текущей clean-работе отдельная multi-class модель не входит в основной pipeline: фокус сделан на качестве binary-модели, а классификация типа риска рассматривается как следующий этап развития проекта.

## 2. Данные и leakage-control

Датасет подготовлен на этапе ДЗ №4 на основе русскоязычного новостного корпуса RuNews / `IlyaGusev/ru_news`. Из общей базы были выделены новости с банковскими упоминаниями, нормализованы названия банков и сформированы текстовые фрагменты вокруг найденных сущностей.

В датасете уже есть готовый `train/valid/test split`, поэтому новое случайное разбиение в ДЗ №5-6 не выполняется. Это важно для сопоставимости экспериментов и контроля утечек.

Поле `event_group_id` используется для leakage-control: один и тот же инфоповод или близкие фрагменты не должны попадать одновременно в обучение и проверку. При этом `event_group_id` не используется как feature.

Поле `found_risk_keywords` также не используется как feature, потому что оно связано с правиловой логикой поиска риск-слов и может создать leakage.

## 3. Последовательность моделей

Моделирование построено как последовательное усложнение:

1. `TF-IDF word`
2. `TF-IDF char`
3. `Combined TF-IDF`
4. `Tuned combined TF-IDF`
5. `LLM-distilled TF-IDF (teacher: GPT-4.1)`
6. `LLM-distilled TF-IDF (teacher: GPT-5.5)`

Word-level TF-IDF учитывает отдельные слова и устойчивые словосочетания. Char-level TF-IDF помогает учитывать морфологию, окончания и варианты написания. Combined TF-IDF объединяет оба представления текста, а tuned-версия дополнительно подбирает параметры `LogisticRegression`.

После сильной классической модели проверяется advanced-подход — LLM distillation. В нём большая языковая модель используется как teacher для подготовки train soft labels, а compact student-модель остаётся быстрой и воспроизводимой: `Combined TF-IDF + LogisticRegression`. После обучения student-модель не требует LLM-teacher на этапе inference.

## 4. Valid-сравнение моделей

Основная метрика выбора модели — `F1` на valid. Она балансирует Precision и Recall, что важно при умеренном дисбалансе классов: accuracy может выглядеть приемлемо, даже если модель плохо находит риск-сигналы.

| Модель | Precision | Recall | F1 | ROC-AUC | PR-AUC |
| ------ | --------: | -----: | -: | ------: | -----: |
| LLM-distilled TF-IDF (teacher: GPT-4.1) | 0.784 | 0.763 | 0.773 | 0.900 | 0.811 |
| LLM-distilled TF-IDF (teacher: GPT-5.5) | 0.689 | 0.816 | 0.747 | 0.895 | 0.805 |
| Tuned combined TF-IDF | 0.771 | 0.711 | 0.740 | 0.872 | 0.790 |
| Combined TF-IDF | 0.730 | 0.711 | 0.720 | 0.859 | 0.765 |
| TF-IDF char | 0.722 | 0.684 | 0.703 | 0.842 | 0.734 |
| TF-IDF word | 0.703 | 0.684 | 0.693 | 0.846 | 0.752 |

Лучший valid F1 среди всех сравниваемых кандидатов показала модель `LLM-distilled TF-IDF (teacher: GPT-4.1)`: F1 = 0.773. Этот результат используется как diagnostic/reference comparison для LLM distillation.

Основная финальная модель clean pipeline — `Tuned combined TF-IDF`. Она фиксируется как воспроизводимое classical ML решение: не зависит от внешнего LLM-teacher ни при обучении, ни на этапе inference, быстро обучается и остаётся сильной reference-моделью.

`GPT-5.5` на valid показала более высокий Recall, но более низкий Precision и F1 по сравнению с `GPT-4.1`. Это означает, что более мощная teacher-модель не дала автоматического прироста по основной метрике выбора.

## 5. LLM distillation

В LLM distillation teacher-модель возвращает два значения:

```text
label
confidence
```

`label` — бинарная оценка риска, а `confidence` отражает уверенность teacher-модели. Далее confidence преобразуется в soft label — вероятность принадлежности объекта к классу `alert_flag = 1`.

Student-модель обучается на смеси исходной hard-разметки и teacher-сигнала:

```text
soft_y = (1 - alpha) * y_train + alpha * teacher_soft_proba
```

Параметр `alpha` регулирует силу влияния teacher-сигнала. Параметр `C` задаёт регуляризацию `LogisticRegression`. Оба параметра выбираются только на valid по F1. Test не используется для выбора teacher-модели, `alpha`, `C` или threshold.

Для GPT-5.5 использовался Responses API с `reasoning={"effort": "low"}`, увеличенным `max_output_tokens` и structured JSON output. Это позволило получить валидные teacher labels для всех train-объектов и контролировать формат ответа.

В LLM distillation teacher-модель используется только на этапе подготовки teacher labels / soft labels для train-выборки. После обучения student-модель не требует LLM-teacher на этапе inference: в production работает компактная `Combined TF-IDF + LogisticRegression`.

## 6. Выбор финальной модели

По актуальным расчётам clean notebook лучший valid F1 показала LLM-distillation модель:

```text
LLM-distilled TF-IDF (teacher: GPT-4.1)
```

Её valid-метрики:

| Precision | Recall | F1 | ROC-AUC | PR-AUC |
| --------: | -----: | -: | ------: | -----: |
| 0.784 | 0.763 | 0.773 | 0.900 | 0.811 |

Однако основной финальной моделью для clean pipeline фиксируется classical ML модель:

```text
Tuned combined TF-IDF
```

Её valid-метрики:

| Precision | Recall | F1 | ROC-AUC | PR-AUC |
| --------: | -----: | -: | ------: | -----: |
| 0.771 | 0.711 | 0.740 | 0.872 | 0.790 |

LLM-distillation остаётся advanced/reference experiment. Test-выборка не используется для переизбрания финальной модели. Она остаётся независимой финальной проверкой и диагностическим сравнением.

## 7. Test-сравнение

На test сравниваются основная финальная classical ML модель и reference-only варианты distillation с teacher `GPT-4.1` и `GPT-5.5`.

| Модель | Precision | Recall | F1 | ROC-AUC | PR-AUC |
| ------ | --------: | -----: | -: | ------: | -----: |
| LLM-distilled TF-IDF (teacher: GPT-5.5) | 0.795 | 0.921 | 0.854 | 0.959 | 0.925 |
| Tuned combined TF-IDF | 0.861 | 0.816 | 0.838 | 0.967 | 0.940 |
| LLM-distilled TF-IDF (teacher: GPT-4.1) | 0.906 | 0.763 | 0.829 | 0.967 | 0.934 |

На test лучшее значение F1 и самый высокий Recall показала `LLM-distilled TF-IDF (teacher: GPT-5.5)`: F1 = 0.854, Recall = 0.921. Это означает, что в диагностическом test-сравнении GPT-5.5 teacher дал наиболее recall-oriented student-модель.

`Tuned combined TF-IDF` остаётся основной финальной моделью: её F1 близок к лучшему test-результату, а Precision и PR-AUC выше, чем у GPT-5.5 distillation. Дополнительно `Tuned combined TF-IDF` показывает самый высокий PR-AUC на test. Это означает, что модель хорошо ранжирует риск-сигналы по вероятности и остаётся сильным решением для сценария, где threshold может подбираться под допустимую нагрузку аналитиков.

Важно: результат GPT-5.5 на test не используется для переизбрания модели задним числом. Он показывает перспективность LLM distillation, но для строгого подтверждения преимущества нужен дополнительный holdout или больше размеченных данных.

## 8. Ограничения

- Датасет небольшой, поэтому Precision, Recall и F1 чувствительны к нескольким объектам.
- В задаче есть дисбаланс между отрицательным и положительным классом.
- Качество LLM distillation зависит не только от мощности teacher-модели, но и от prompt-инструкции, confidence calibration, качества исходной разметки и профиля ошибок teacher.
- Различие между valid и test результатами нужно интерпретировать осторожно: test не должен использоваться для обратного выбора модели.

## 9. Что улучшать дальше

- Расширить размеченную выборку.
- Вручную проверить false positives и false negatives.
- Уточнить `event_group_id` для более строгого контроля near-duplicates.
- Проверить стабильность качества на новом holdout.
- Подобрать threshold под реальную нагрузку аналитиков.
- Отдельно развить multi-class классификацию `risk_type_4cls` как следующий этап, не смешивая её с текущей clean binary-работой.
