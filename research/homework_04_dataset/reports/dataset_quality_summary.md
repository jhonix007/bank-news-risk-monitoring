# Краткий отчёт о качестве датасета

| Проверка | Результат | Риск | Что сделать |
| -------- | --------: | ---- | ----------- |
| Размер датасета | 839 строк | Небольшой объём ограничивает сложные модели | Начать с TF-IDF baseline |
| Баланс `alert_flag` | 585 negative / 254 positive | Accuracy может быть завышенной | Использовать precision, recall, F1, PR-AUC |
| Баланс `risk_type_4cls` | `no_risk` 585, `cyber_risk` 111, `operational_risk` 97, `regulatory_risk` 46 | Multi-class метрики по малым классам нестабильны | Основной задачей считать binary `alert_flag` |
| Пропуски | Есть ограниченное число пустых значений в неключевых полях | Vectorizer не принимает `NaN` | В pipeline использовать `fillna('')` |
| Дубликаты `sample_id` | 0 | Нет риска ID-дублей | Контроль сохранять |
| Exact duplicates `title + text_fragment + entity_norm` | 0 | Прямое запоминание маловероятно | Дополнительно проверять near-duplicates |
| Пересечение `event_group_id` между split | 0 | Group leakage снижен | Использовать текущий split |
| Exact duplicates между split | Проверяется в notebook | Возможен leakage при текстовых дублях | Не обучать без проверки split |
| Near-duplicates между split | Проверяется TF-IDF word/char similarity | Возможны близкие пересказы одного инфоповода | В следующей версии строить расширенный `event_cluster_id` |
| Leakage features | Есть технические и target-связанные колонки | Модель может получить подсказку | Исключить `sample_id`, `event_group_id`, `split`, `risk_type_4cls`, `found_risk_keywords` |
| Пригодность для binary baseline | Пригоден | Ограничения по размеру, источникам и периоду | Обучать простой baseline и анализировать ошибки |
| Пригодность для multi-class `risk_type_4cls` | Условно пригоден | Классы риска меньше, особенно `regulatory_risk` | Обучать как дополнительную задачу, использовать macro F1 |

## Итог

### A. Можно ли обучать бинарную модель?

Да. Датасет пригоден для учебного бинарного baseline `alert_flag`, если использовать group-aware split и исключить leakage-признаки.

### A2. Можно ли обучать многоклассовую модель?

Да, можно обучать отдельную multi-class модель с target `risk_type_4cls`. Это дополнительная задача: модель будет предсказывать один из классов `no_risk`, `cyber_risk`, `operational_risk`, `regulatory_risk`. Метрики нужно считать через macro F1 и смотреть качество по каждому классу отдельно, потому что классы риска меньше, чем `no_risk`.

### B. Какие ограничения?

Датасет небольшой, исторический и собран из ограниченного набора источников. Возможны source bias, temporal drift и остаточные near-duplicates.

### C. Что улучшить в следующей версии?

Нужно расширить свежими новостями, добавить проверку вторым разметчиком, улучшить группировку похожих инфоповодов до `event_cluster_id` и отдельно добрать редкие риск-классы.
